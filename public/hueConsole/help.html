About the Hue Console

This is a project to try out the Philips Hue API in the AngularJS web environment and learn a little bit about color models and the 'Internet of Things'.

FAQ

What is Hue?
Hue is a system of programmable light bulbs from Philips. http://www2.meethue.com/en-us/what-hue-does/ambience/. It's a great way to get started with smart home automation, the internet of things, REST APIs, and web and mobile development without worrying about turning your heat up to 120 degrees and unlocking all your doors at 3 a.m. whenever your code throws an exception :)

I became interested in light because of the universal appeal. Not everyone is into wearables, like Google Glass, and not everyone wants a smart coffee-maker, smart dog collar, or JavaScript-controlled helicoptor, but everyone in the world, regardless of age, lifestyle, career, ethnicity, geo-location, or any other demographic factor, needs, loves, and seeks light--especially quality, comforting light. Programmable lights are something the whole family can appreciate and benefit from, so I thought it was a great place to start when looking at the 'Internet of Things.'

What does this app do that the stock phone app for Hue doesn't do?
One thing to point out is that this client sets color temperature. Most apps, including the official one from Philips, don't, and color temperature is very important for mood, psychology, and simply matching the decor, time of day, and other light sources in your home.

Can I use this site from my phone or tablet?
Yes, but you'll need to host it locally somewhere on the same network as your hue bridge or emulator, such as your laptop or other small web-server. You can also run it from http://stevenvictor.net/hueConsole/ (assuming your browser is on the same network of your hue hub). Here is a video demo if you can't run the app locally.

Then why didn't you make it a phone app? :)
I may in the future -- this was just for fun.

What is this "register/add user" button?
Hue has some built-in security to prevent unauthorized use. It's nothing as sophisticated as OAuth, but it's reasonable for now. Aside from being on the same local network, the out-of-the-box Hue needs to be configured to work with each new client (official philips app, other apps, this app, etc...). To register your client app, you must press the 'link' button on the actual hue bridge device and then click the 'register/add user' button on the main page. This will register a client with the name specified in the 'Client-Id' field. 'click register ->' will be replaced by a hexadecimal app id. See http://www.developers.meethue.com/documentation/getting-started/ for details

Why do some text and areas on the page display with red strikethrough?
While you can set any parameter on Hue that you want (hue, saturation, color temperature, etc...), not all values apply at all times. For example, since color temperature only corresponds to whiteish light values with a reddish to blueish cast, many colors, such as a deep intense lime green, don't have a corresponding color temperature value, at least not on the Hue system. Therefore, if you set your light colors, the temperature value is shown with strikeout to show that is currently doesn't apply. It is possible to convert color temperature to an RGB value preview, but I'm not sure what this conversion formula is, and I have heard it is inherently inaccurate, so I don't currently support it. I could use the Hue system to help calculate HSB (and then derive RGB) from color temperature, but that would cause a fair amount of network lag, so I'd rather not support that either.

I have more than one Hue light -- can I set indiviual light colors/states?
Not currently. I didn't want to add too much UI to my first Hue app. If I create another app with a narrower focus (just color temperature), I might add support for individual lights or even scenes.
